*lazyvim-cheatsheet*        LazyVim key cheat sheet

LazyVim Key Cheat Sheet                       *lazyvim-cheatsheet-title*

이 문서는 이 설정 repo에서 자주 쓰는 키만 작업 흐름별로 정리한다.
`<leader>`는 Space 키다. 모르는 키가 있으면 Space를 누른 뒤 잠깐
기다리면 which-key가 가능한 키 목록을 보여준다.

==============================================================================
빠른 시작                                             *lazyvim-cheatsheet-start*

키                      동작
------------------------------------------------------------------------------
Space                  leader 키. 다음 키 후보 보기
<leader>?              현재 버퍼의 키맵 검색
<leader>h?             로컬 help 문서 목록 열기
:HelpDocs              로컬 help 문서 목록 열기
<C-s>                  저장
j / k                  긴 줄 기준 화면 줄 단위 이동
jk                     Insert 모드 종료
<Esc>                  검색 highlight 지우기, 일반 종료 동작
<leader>l              Lazy 플러그인 관리자
<leader>L              LazyVim changelog

파일과 검색                                            *lazyvim-cheatsheet-file*

키                      동작
------------------------------------------------------------------------------
<leader>e              Explorer 열기(root 기준)
<leader>E              Explorer 열기(cwd 기준)
<leader>ff             파일 찾기
<leader>fF             현재 cwd 기준 파일 찾기
<leader>fg             Git tracked 파일 찾기
<leader>fr             최근 파일
<leader>fb             버퍼 목록
<leader>fp             Project 목록(tmux-sessionizer dirs 기준)
<leader>fP             Lazy 플러그인 파일 찾기
<leader>fn             새 파일
<leader>sg             Grep 검색
<leader>sw             커서 단어 검색
<leader>sW             커서 WORD 검색
<leader>sb             현재 버퍼 검색
<leader>st             TODO 검색
<leader>sT             TODO/FIX/FIXME 검색
n / N                  다음/이전 검색 결과로 이동, 화면 중앙 유지

Tip:
  정확한 검색 키가 기억나지 않으면 `<leader>s`까지 누르고 which-key를 본다.

Explorer:
  <CR> / l             파일 열기 또는 디렉토리 toggle
  -                    파일을 horizontal split으로 열기
  |                    파일을 vertical split으로 열기
  M                    Explorer 최대화 toggle(Shift+m)
  h                    디렉토리 닫기
  <BS>                 상위 디렉토리로 이동
  a                    파일/디렉토리 추가
  r                    파일 rename
  c                    선택 파일을 현재 디렉토리에 복사, 단일 파일은 새 이름으로 복사
  C                    현재/선택 파일을 같은 위치에 .copy 이름으로 복제
  m                    선택 파일을 현재 디렉토리로 이동, 단일 파일은 rename
  y / p                파일 경로 yank / register의 파일 복사 붙여넣기
  d                    삭제
  H / I                hidden / ignored 파일 toggle
  T                    파일 상세 정보(크기/수정 시각) toggle(Shift+t)
  <C-s> / <C-v>        Picker 기본 horizontal / vertical split 열기

Git                                                     *lazyvim-cheatsheet-git*

키                      동작
------------------------------------------------------------------------------
<leader>gg             GitUI 열기, repo root 기준
<leader>gG             GitGraph 커밋 그래프 열기
<leader>gl             Git log
<leader>gL             Git log, 현재 cwd 기준
<leader>gf             현재 파일 Git history
<leader>gB             현재 파일/선택 영역을 브라우저에서 열기
<leader>gY             Git URL 복사
<leader>gd             Diffview 열기
<leader>gh             현재 파일 Diffview history
<leader>gH             브랜치/전체 Diffview history
<leader>gq             Diffview 닫기
<leader>gb             현재 라인 blame toggle
<leader>gp             현재 hunk 미리보기
<leader>gs             현재 hunk stage
<leader>gS             현재 buffer stage
<leader>gu             hunk stage 취소
<leader>gr             현재 hunk reset
<leader>gR             현재 buffer reset
]h / [h                다음/이전 hunk
]H / [H                마지막/첫 hunk

Octo (GitHub, gh CLI 인증 필요):
<leader>gop            PR 목록
<leader>goP            PR 검색
<leader>goi            Issue 목록
<leader>goI            Issue 검색
<leader>gor            Repo 목록
<leader>gos            GitHub 검색
PR/Issue 버퍼 안에서 <localleader> 그룹으로 코멘트/리뷰/머지

GitLab (사내 self-hosted, GITLAB_TOKEN/GITLAB_URL 필요):
<leader>gmm            MR 목록에서 선택 (checkout + 리뷰 대상 지정)
<leader>gmr            현재 브랜치 MR 리뷰 (diff)
<leader>gms            MR summary
<leader>gmp            파이프라인 보기
<leader>gma            MR 승인
<leader>gmA            승인 취소
<leader>gmc            MR 생성

GitLab 화면 닫기/조작:
  <leader>gq            리뷰 diff 닫기 (Diffview 기반)
  gld                   discussion 패널 토글
  :q                    summary/pipeline 팝업 닫기
  ZZ / ZQ               편집 팝업 제출 / 버리고 닫기
  g?                    현재 화면 키맵 도움말
  gl* 전역 키맵도 플러그인이 등록 (glc, gls, glA, glM 등)

GitGraph 안에서:
  Enter                 커서의 commit을 Diffview로 열기
  Visual 선택 + Enter   선택한 commit 범위를 Diffview로 열기

Diffview 안에서:
  <leader>gq            DiffviewClose
  q                     일부 panel/window 닫기

외우는 순서                                           *lazyvim-cheatsheet-order*

1. Space를 눌러 which-key 보기
2. 파일/검색: `<leader>f`, `<leader>s`
3. Git: `<leader>g`
4. 코드/LSP: `<leader>c`, `gd`, `gr`, `K`
5. 문제 확인: `<leader>x`, `[d`, `]d`
6. UI toggle: `<leader>u`
